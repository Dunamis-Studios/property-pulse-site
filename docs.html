Setup Guide

Install Property Pulse, configure tracked properties, and start seeing live property history on every record in minutes.

Requirements

  • A HubSpot account with access to CRM records (any tier that supports CRM customization and HubDB)
  • Admin or App Marketplace permissions in your HubSpot portal to install the app

Step 1: Install the App

1

Click the Install button on the HubSpot Marketplace listing for Property Pulse. HubSpot will prompt you to choose the portal to install the app into.

2

Review the requested permissions and click Connect app. Property Pulse requires read access to CRM records, schemas, and owners across Contacts, Companies, Deals, Tickets, and Custom Objects in order to fetch property metadata and change history. It also needs HubDB access to store the admin-configured list of tracked properties inside your portal.

3

Once authorized, the Property Pulse tab will appear on every supported CRM record type. It will start empty until you configure tracked properties in Step 2.

Step 2: Configure Tracked Properties

1

Open any CRM record that has the Property Pulse tab. In the bottom-right corner of the card, click the Settings gear to open the Property Pulse settings page.

2

The settings page shows one collapsible accordion per supported object type -- Contacts, Companies, Deals, Tickets, plus one for each custom object in your portal. Expand the object type you want to configure.

3

Use the Add a property to track dropdown at the bottom of the section to pick a property from the object type's full property list. Click Add to add it. Repeat for every property you want to track.

4

Reorder tracked properties using the ↑ and ↓ buttons on each property card. The order here is the order the tiles will appear on the record.

Step 3: Per-Section Settings

Each object type section has two global controls:

  • Record layout columns -- how many columns of tiles to render on the record page. 1 column gives each property a full-width tile; 2 or 3 columns tile them side by side for a denser layout.
  • Allow users to add their own properties -- when on, individual users can add their own tracked properties in addition to the admin-configured set. See "User-added properties" below.

Step 4: Per-Property Settings

Each tracked property card exposes three display options that control how that specific property's tile renders on the record:

  • Log -- when on, shows a paginated change log table beneath the current value, with one row per property change.
  • Source -- when on, adds a Source column to the change log table showing where each edit came from (CRM UI, Workflow, API, Integration, Import, etc.) and, for manual edits, the user who made the change.
  • Card size -- Full uses standard padding on the record tile. Compact reduces padding for a tighter layout when tracking many properties.

Step 5: Save

Use the Save {Object Type} settings button at the bottom of each section to save changes to that section only, or the Save all changes button at the bottom of the page to save every section with unsaved changes at once. Sections with no changes are skipped.

Note: Settings are stored inside your own HubSpot portal as a HubDB table called property_pulse_config_v2. Nothing is stored on any external server.

How the Card Renders on a Record

Once configured, every supported record gets a Property Pulse tab that shows one tile per tracked property. Each tile displays:

  • The property's display name and a badge indicating whether it was admin-tracked or added by the current user
  • The current value and -- if there's a prior entry -- the previous value with a relative timestamp
  • A structured, sortable, paginated change log table with Property Value, Source, and Date columns. Owner-type properties automatically resolve numeric owner IDs to user names (e.g. "Joshua Bradford" instead of "26447425")

User-Added Properties

If the admin enables Allow users to add their own properties for an object type, individual users can track additional properties beyond the admin-configured set. User-added properties:

  • Appear in a separate "Your additional properties" section on the record, underneath the admin-tracked section
  • Are scoped to the current user only -- other users in the portal will not see them
  • Apply across every record of that object type, so the user only has to add the property once
  • Can be removed by the user at any time via the Remove button on the tile -- removal also only affects that user's view

Troubleshooting

The card shows "No recorded changes for this property"

HubSpot only retains property history for properties that have been edited since tracking started. If a property has never changed on a record, there is nothing for Property Pulse to display. Make an edit to the property and refresh the record.

An owner property shows a raw numeric ID

Property Pulse resolves owner IDs by calling the HubSpot owners API. If the ID doesn't match any active owner -- for example, if the owner was deleted or never existed -- the tile falls back to rendering "Owner <id>" so you can still see the value.

Settings page is empty after installing

The first time you open the settings page, Property Pulse creates the HubDB table it uses for configuration storage. If this fails, confirm that HubDB is enabled for your portal (requires CMS Hub Professional or higher, or Marketing Hub Professional or higher).
