Setup Guide

Add the Property Pulse card to your record views, configure tracked properties per object type, and start seeing live property history on every record in minutes.

Requirements

  • A HubSpot account with access to CRM records (any tier that supports CRM customization and HubDB)
  • Super admin permissions (or App Marketplace permissions) on the HubSpot portal you want to install into -- only a user with those permissions can approve the OAuth scope consent screen on behalf of the account

Step 1: Install and Authorize the App

Property Pulse uses OAuth for installation. You'll approve a set of read-only scopes on a HubSpot-hosted consent screen, and HubSpot will connect your portal to the app. No private app token, no client secret, no manual credential entry.

1

From the HubSpot Marketplace listing for Property Pulse, click Install app. HubSpot will prompt you to pick which portal to install into if you have access to more than one.

2

Review the scopes Property Pulse is requesting on the consent screen. The app asks for read access to contact, company, deal, ticket, custom object, and owner records and schemas, plus HubDB access for storing tracked-property configuration inside your portal. There are no write scopes for CRM records -- Property Pulse cannot modify your data.

3

Click Connect app. HubSpot redirects to the Property Pulse OAuth callback, which exchanges the authorization code for a refresh token and stores it against your portal ID. You'll see a confirmation page that the app was installed successfully.

Note: You can revoke Property Pulse's access at any time from Settings → Integrations → Connected Apps → Property Pulse → Uninstall. Uninstalling invalidates the stored refresh token immediately.

Step 2: Add Property Pulse to a Record View

Installing the app makes the Property Pulse card available in the card library, but it won't automatically appear on any record. You need to add it to a record view for each object type you want to use it on.

1

In HubSpot, navigate to any CRM record page for the object type you want to add the card to -- a Contact, Company, Deal, Ticket, or any Custom Object record.

2

In the top right of the record view, click Customize.

3

Decide where the Property Pulse card should live. You can add it to an existing tab (such as the Overview tab) or create a new dedicated tab for it. If you're tracking more than a handful of properties, a dedicated tab keeps the rest of the record view uncluttered.

4

If you're creating a new tab, add it first and give it a name -- for example, Property Pulse or Property History.

5

In the tab editor, open the card library on the left and search for Property Pulse to find the card.

6

Drag the card onto the tab layout, or click it to add it. Position it wherever you want it to appear.

7

Save the customization. The Property Pulse card will now appear on every record of that object type. It will start empty until you configure tracked properties in Step 3.

Note: Record view customization is per object type. If you want Property Pulse on Contacts and on Companies, repeat this process once from a Company record as well. Each object type has its own record view layout.

Step 3: Configure Tracked Properties

1

Open any CRM record where you added the Property Pulse card in Step 2. In the bottom-right corner of the card, click the Settings gear to open the Property Pulse settings page.

2

The settings page shows one collapsible accordion per supported object type -- Contacts, Companies, Deals, Tickets, plus one for each custom object in your portal. Expand the object type you want to configure.

3

Use the Add a property to track dropdown at the bottom of the section to pick a property from the object type's full property list. Click Add to add it. Repeat for every property you want to track.

4

Reorder tracked properties using the ↑ and ↓ buttons on each property card. The order here is the order the tiles will appear on the record.

Step 4: Per-Section Settings

Each object type section has two global controls:

  • Record layout columns -- how many columns of tiles to render on the record page. 1 column gives each property a full-width tile; 2 or 3 columns tile them side by side for a denser layout.
  • Allow users to add their own properties -- when on, individual users can add their own tracked properties in addition to the admin-configured set. See "User-added properties" below.

Step 5: Per-Property Settings

Each tracked property card exposes three display options that control how that specific property's tile renders on the record:

  • Log -- when on, shows a paginated change log table beneath the current value, with one row per property change.
  • Source -- when on, adds a Source column to the change log table showing where each edit came from (CRM UI, Workflow, API, Integration, Import, etc.) and, for manual edits, the user who made the change.
  • Card size -- Full uses standard padding on the record tile. Compact reduces padding for a tighter layout when tracking many properties.

Step 6: Save

Use the Save {Object Type} settings button at the bottom of each section to save changes to that section only, or the Save all changes button at the bottom of the page to save every section with unsaved changes at once. Sections with no changes are skipped.

Note: Tracked-property settings are stored inside your own HubSpot portal as a HubDB table called property_pulse_config_v2. No CRM data or settings are stored externally. The only thing Property Pulse stores outside your portal is a HubSpot OAuth refresh token, kept on the app's backend server so the app can authenticate against HubSpot APIs on your behalf. See the privacy policy for full details.

How the Card Renders on a Record

Once configured, every record of an object type that has the Property Pulse card added to its view will show one tile per tracked property inside the card. Each tile displays:

  • The property's display name and a badge indicating whether it was admin-tracked or added by the current user
  • The current value and -- if there's a prior entry -- the previous value with a relative timestamp
  • A structured, sortable, paginated change log table with Property Value, Source, and Date columns. Owner-type properties automatically resolve numeric owner IDs to user names (e.g. "Joshua Bradford" instead of "26447425")

User-Added Properties

If the admin enables Allow users to add their own properties for an object type, individual users can track additional properties beyond the admin-configured set. User-added properties:

  • Appear in a separate "Your additional properties" section on the record, underneath the admin-tracked section
  • Are scoped to the current user only -- other users in the portal will not see them
  • Apply across every record of that object type, so the user only has to add the property once
  • Can be removed by the user at any time via the Remove button on the tile -- removal also only affects that user's view

Troubleshooting

The card shows "No recorded changes for this property"

HubSpot only retains property history for properties that have been edited since tracking started. If a property has never changed on a record, there is nothing for Property Pulse to display. Make an edit to the property and refresh the record.

An owner property shows a raw numeric ID

Property Pulse resolves owner IDs by calling the HubSpot owners API. If the ID doesn't match any active owner -- for example, if the owner was deleted or never existed -- the tile falls back to rendering "Owner <id>" so you can still see the value.

Settings page is empty after installing

The first time you open the settings page, Property Pulse creates the HubDB table it uses for configuration storage. If this fails, confirm that HubDB is enabled for your portal (requires CMS Hub Professional or higher, or Marketing Hub Professional or higher).
