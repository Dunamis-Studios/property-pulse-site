Privacy Policy

Last updated: April 15, 2026

This Privacy Policy describes how Dunamis Studios ("we", "us", "our") handles information when you use Property Pulse ("the App").

1. What We Collect

We do not collect, store, or retain any of your CRM data on any external server. Property data, property change history, and per-record details are read from HubSpot's APIs in real time on each request and are never persisted outside your portal. The only thing the App stores outside your portal is a HubSpot OAuth refresh token (described below), which is the minimum required to let the App authenticate against HubSpot APIs on your behalf when you use it.

The App accesses the following CRM data only in real time, only for the duration of a single page load, and only to render the Property Pulse card or settings page:

  • CRM property definitions for the object type being viewed
  • Property history entries for tracked properties on the record being viewed
  • Owner directory entries (to resolve owner IDs and resolve the author name on manual property changes)
  • Custom object schemas (to discover which custom object types exist in your portal)

None of this data is logged, cached beyond the lifetime of an individual request, or persisted on any Dunamis Studios server.

2. OAuth Tokens

Property Pulse installs into your portal via HubSpot's OAuth flow. When you approve the install, HubSpot issues the App a pair of tokens:

  • A short-lived access token (~30 minute lifetime) used to authenticate HubSpot API calls on your behalf.
  • A longer-lived refresh token used to obtain a new access token when the current one expires.

The refresh token for your portal is stored on the App's backend server (hosted on Vercel) in a Redis database, keyed by your HubSpot portal ID. This is the only thing the App persists outside your portal. Access tokens are held only in memory for the duration of a single request and are not written to any persistent store.

The refresh token is never exposed to any third party, never logged, and is protected by the HubSpot OAuth scopes your admin approved on install. Property Pulse only requests read-only scopes for CRM objects and schemas, plus HubDB access for storing tracked-property configuration inside your own portal -- it cannot modify your CRM records.

3. Where Settings Are Stored

Admin-configured tracked properties and per-property display settings are stored inside your own HubSpot portal as a HubDB table named property_pulse_config_v2. Users' personal tracked property additions are stored in the same table, keyed by HubSpot user ID so each user only sees their own selections.

Because the configuration table lives entirely inside your portal, you own the data and can inspect or delete it at any time from the HubDB area of HubSpot. No copy of this configuration is kept on the App's backend server.

4. Data Sharing

We do not sell, rent, or share any data with third parties. The App does not contain any third-party analytics, tracking pixels, or telemetry. The refresh token stored on the backend is used solely to make authorized API requests to HubSpot on your behalf.

5. Data Retention

When you uninstall Property Pulse from your portal, the HubSpot OAuth refresh token associated with your portal is invalidated immediately by HubSpot, and the App deletes its stored copy from Redis on next access (or you can request immediate deletion at the contact email below).

The property_pulse_config_v2 HubDB table remains in your portal after uninstall so you don't lose your tracked property configuration if you ever reinstall. You can delete it manually from the HubDB area of HubSpot at any time.

6. Security

Communication between the App's HubSpot UI extension and the backend server is authenticated using HubSpot's signed-request protocol: every request the UI extension makes is signed with the app's client secret, and the backend rejects any request whose signature doesn't validate. All traffic between the UI extension, the backend, and HubSpot APIs runs over HTTPS.

Refresh tokens are stored in a managed Redis instance accessible only to the backend server's authenticated clients. The HubSpot OAuth client secret lives only as an environment variable on the backend server and is never sent to the browser or stored in client-side code.

7. Your Rights

You may revoke the App's access to your HubSpot portal at any time by uninstalling it from Settings → Integrations → Connected Apps. Uninstalling immediately invalidates the stored OAuth tokens and ends all API access. To additionally request deletion of your backend-stored refresh token record before it's purged on next access, contact us at the email below.

8. Changes to This Policy

We may update this Privacy Policy from time to time. We will post the updated policy on this page with a revised date.
